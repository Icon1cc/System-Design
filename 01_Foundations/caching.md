**Code Example (Python):**
```python
def get_user(user_id):
    # 1. Try cache first
    cached = cache.get(f"user:{user_id}")
    if cached:
        return cached  # Cache hit!

    # 2. Cache miss - query database
    user = database.query(f"SELECT * FROM users WHERE id = {user_id}")

    # 3. Store in cache for next time
    cache.set(f"user:{user_id}", user, ttl=300)  # 5 min TTL

    return user

def update_user(user_id, data):
    # 1. Update database
    database.update(f"UPDATE users SET ... WHERE id = {user_id}")

    # 2. Invalidate cache
    cache.delete(f"user:{user_id}")
```

## Interview Questions

### Question 1: "How do you decide what to cache?"

**Strong Answer:**
```
I cache data that is:

1. FREQUENTLY ACCESSED
   â€¢ High read-to-write ratio
   â€¢ Popular items, user sessions, config data

2. EXPENSIVE TO COMPUTE/FETCH
   â€¢ Complex database queries
   â€¢ Aggregations
   â€¢ External API calls

3. TOLERANT OF STALENESS
   â€¢ Product catalogs (okay if slightly stale)
   â€¢ NOT: bank balances, inventory counts

4. SIZED APPROPRIATELY
   â€¢ Fits in available memory
   â€¢ Not so large it evicts other useful data

I would NOT cache:
â€¢ Rapidly changing data
â€¢ User-specific data with low access (cold data)
â€¢ Security-sensitive data that shouldn't persist
â€¢ Data that must always be fresh
```

### Question 2: "Explain cache invalidation strategies"

**Strong Answer:**
```
Main strategies:

1. TTL (Time-Based)
   â€¢ Set expiration time on cache entries
   â€¢ Simple, but may serve stale data
   â€¢ Good for: product listings, news feeds

2. Event-Based (Write-Through)
   â€¢ Invalidate/update cache on every write
   â€¢ Always fresh, but more complex
   â€¢ Good for: user profiles, settings

3. Hybrid
   â€¢ Short TTL + event-based invalidation
   â€¢ Best of both worlds
   â€¢ Cache refreshes periodically AND on updates

Key considerations:
â€¢ What's the acceptable staleness?
â€¢ What's the write frequency?
â€¢ What's the cost of serving stale data?

â€¢ Product info: TTL 1 hour (low write frequency)
â€¢ Inventory count: Event-based (critical accuracy)
â€¢ Reviews: TTL 5 min (acceptable lag)
```
